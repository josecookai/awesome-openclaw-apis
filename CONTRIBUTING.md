# Contributing to Awesome OpenClaw

感谢你的贡献！🎉

## 如何贡献

### 1. 报告问题
- 使用 GitHub Issues
- 描述清楚问题或建议
- 提供复现步骤（如果是 bug）

### 2. 提交新内容

#### 对于 Skills 仓库

**新增技能 PR 模板:**

```markdown
## 新增技能: [技能名称]

### 技能信息
- **名称**: 
- **描述**: （一句话描述）
- **安装命令**: `clawdhub install [name]`
- **官方链接**: 
- **分类**: [核心必备/生产力/开发工具/社交媒体/数据分析/多媒体/生活服务]

### 使用示例
```bash
# 基础用法
clawdhub install [name]

# 示例场景
```

### 测试环境
- [ ] OpenClaw 版本: 
- [ ] 操作系统: 
- [ ] 测试日期: 

### 检查清单
- [ ] 技能已安装并测试
- [ ] 链接可访问
- [ ] 分类正确
- [ ] 遵循格式规范

### 附加信息
- 是否有已知问题?
- 是否需要特殊配置?
- 推荐的使用场景?
```

#### 对于 APIs 仓库

**新增 API PR 模板:**

```markdown
## 新增 API: [API名称]

### 基本信息
- **名称**: 
- **类型**: [AI模型/搜索/生产力/社交/开发/多媒体/生活服务]
- **官网**: 
- **定价**: [免费/按量付费/订阅]

### 注册步骤
1. 访问 [注册链接]
2. ...
3. 获取 API Key

### 环境变量
```bash
export API_NAME="your-key"
```

### 使用示例
```bash
# 示例代码
```

### 测试状态
- [ ] 已注册
- [ ] 已测试
- [ ] 已配置环境变量

### 检查清单
- [ ] API 稳定运行
- [ ] 有明确文档
- [ ] 定价透明
- [ ] 遵循法律法规

### 附加信息
- 地区限制?
- 是否需要信用卡?
- 免费额度?
```

## 贡献流程

1. **Fork** 仓库
2. 创建分支: `git checkout -b add-[skill-name]`
3. 修改文件
4. 提交: `git commit -m "Add [skill-name]"`
5. Push: `git push origin add-[skill-name]`
6. 提交 **Pull Request**

## 格式规范

### Markdown 格式
- 使用标准 Markdown
- 表格对齐
- 链接使用 `[text](url)` 格式

### 排序规则
- 按字母顺序排列（同一分类内）
- 优先级高的放前面

### 描述要求
- 简洁明了（不超过 100 字）
- 说明核心功能
- 提及使用场景

## 审核流程

1. 提交 PR 后，维护者会在 3 天内审核
2. 可能需要修改，请保持关注
3. 通过后合并到主分支

## 行为准则

- 尊重他人
- 建设性反馈
- 遵守法律法规

## 问题?

如有疑问，欢迎开 Issue 讨论！
